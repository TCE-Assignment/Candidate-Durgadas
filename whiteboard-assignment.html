🖍 Whiteboard App Assignment

Platform: Android (Kotlin) | Target: IFP Devices | Local Storage

📌 Objective

Build a whiteboard application using Kotlin designed for Interactive Flat Panels (IFPs). All drawings, shapes, and text interactions must be stored locally in a structured .json format. The application should work offline, support large screens (65″–85″) in landscape orientation, and enable teaching, drawing, and collaboration directly on the screen.

🔗 Reference Design & Functionality

Use the following app for inspiration in layout and interaction design:

👉 https://v0-tata-class-edge.vercel.app/

🛠 Core Functional Requirements

1. ✏ Freehand Drawing

  • Touch and stylus support using Android Canvas.
  • Smooth curves with Path.
  • Track each stroke's:
    • Points
    • Color
    • Stroke width

2. 🧼 Eraser

  • Toggle eraser mode.
  • Pixel-based erasing using touch drag.
  • Optional: highlight erase area.
  • Text deletion should support:
    • Pixel-based partial erasing
    • Full-text selection removal

3. 🎨 Stroke Width & Color

  • Adjustable stroke width via slider or fixed-size buttons (Thin / Medium / Thick).
  • Choose from a palette of 5–6 colors.

4. 🔺 Insert Shapes

  • Shapes supported:
    • Rectangle
    • Circle
    • Line
    • Polygon (4+ sides)
  • Drag to place and resize.
  • Properties to save:
    • Shape type
    • Size and position
    • Color

5. 🔠 Insert Text

  • Tap to insert a text box.
  • Editable text, font size, and color.
  • Movable after insertion.
  • Erasable via both pixel-based and full-selection modes.

6. 💾 Save & Load Locally

  • Save whiteboard state as .json in internal or external app-specific storage.
  • Load JSON files to restore the canvas.
  • Use filenames like:
    whiteboard_yyyyMMdd_HHmmss.json

✅ Example JSON Structure

{
  "strokes": [
    {
      "points": [[10, 10], [15, 20], [20, 30]],
      "color": "#FF0000",
      "width": 3
    }
  ],
  "shapes": [
    {
      "type": "rectangle",
      "topLeft": [50, 50],
      "bottomRight": [150, 100],
      "color": "#0000FF"
    }
  ],
  "texts": [
    {
      "text": "Hello IFP!",
      "position": [300, 400],
      "color": "#000000",
      "size": 24
    }
  ]
}

📱 Device & UX Considerations

  • Designed for Interactive Flat Panels (65″–85″)
  • Landscape-only mode
  • UI elements should scale for large touch displays
  • All files are saved locally
  • No reliance on internet access
  • Use scalable dimensions (dp/sp) and large touch targets

🧰 Tech Guidelines (Android - Kotlin)

  • Custom Drawing: Use Canvas, Paint, and Path
  • State Management: Use ViewModel, LiveData, StateFlow
  • Serialization: Use Gson or kotlinx.serialization for JSON

Project Structure:

/models        → Data classes for strokes, shapes, text
/services      → File handling, JSON loading/saving
/views         → Drawing canvas, toolbars
/viewmodels    → App logic & state
/ui            → Screens and fragments
MainActivity.kt

⚙ Technical Expectations

  • Kotlin best practices
  • Null safety
  • Modular architecture (MVVM)
  • Well-documented code
  • Clean and reusable components
  • Handle configuration changes and scaling gracefully
